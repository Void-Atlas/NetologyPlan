# 1 Перечень автоматизируемых сценариев на сайте Нетологии(https://netology.ru/navigation?utm_campaign=brand_all_n-29519_2026_01_19_day_student_land&utm_source=trigger&utm_medium=block/#order)

### Позитивный сценарии
Сценарий 1. Переход к странице профессии через каталог: Каталог курсов -> программирование -> 
инженер по тестированию -> заполнение формы (Имя, номер телефона, эл. почта)

Сценарий 2. Переход к странице профессии через направление обучения на главной странице сайта: Главная страница ->
программирование -> инженер по тестированию -> заполнение формы (Имя, номер телефона, эл. почта)

Сценарий 3. Переход к странице процессии через рекламный баннер акции сверху на главной странице:
Главная страница -> выбрать курс -> программирование -> инженер по тестированию -> заполнение формы (Имя, номер телефона, эл. почта)

Сценарий 4. Успешное заполнение всех обязательных полей формы валидными данными: Инженер по тестированию (https://netology.ru/programs/qa-middle) ->
форма заполнения.
1) Заполняем поле логина валидными данными.
2) Заполняем поле номера телефона валидными данными.
3) Заполняем поле эл. почты валидными данными.
4) Кликаем на кнопку "Получить консультацию"

### Негативный сценарии

Сценарий 1. Попытка отправки формы с пустыми обязательными полями -> проверка показа ошибок валидации под полями.

Сценарий 2. Некорректный email и телефон → проверка показа ошибок валидации.

Сценарий 3. Проверка поведения при написании имени/телефона/почты свыше 100 символов.

Сценарий 4. Проверка мобильной версии: переход и заполнение формы на разрешении < 768px.


# 2 Перечень используемых инструментов с обоснованием выбора.

1) Selenide WebDrive + Java 11+ 
Обоснование: классика, стабильная работа с динамическими SPA, хорошая поддержка сообществом,
много готовых практик по работе с формами и модалками.
2) JUnit 5+ 
Обоснование: удобные фикстуры, parametrize, хорошая отчётность (особенно с allure/gradlew).
3) Allure — система отчётности
   Обоснование: красивые отчёты с шагами, скриншотами, логами, видео — удобно показывать заказчику/менеджеру.
4) BrowserStack/SauceLabs/Selenoid — облачная ферма браузеров
   Обоснование: если нужно проверить много комбинаций ОС, браузер и версию.
5) IntelliJ IDEA Ultimate - удобный инструмент разработки.
6) Docker Compose (если дадут доступ к БД)
7) Faker для генерации случайных пользователей

# 4 Перечень и описание возможных рисков при автоматизации.

Риск 1. Отправка реальной заявки на продакшен (высокий)
Последствия: спам в CRM, негатив от отдела продаж, блокировка IP.

Риск 2. Изменение структуры сайта / селекторов (средний-высокий)
Последствия: падения тестов после каждого релиза.

Риск 3. Возможны ложные срабатывания тестов. (низкий)
Последствия: проверка ошибок в тесте тратит ресурсы тестирования.

Риск 4. Возможны затруднения по поиску локаторов элементов страницы для тестирования.
Последствия: сроки написания автотестов увеличатся.

# 6 Интервальная оценка с учётом рисков в часах.
Написание автотестов займет около 7-10 часов, если будет писать 1 человек. 2 человека справятся за 3-5 часов в зависимости от выданных разрешений со стороны Нетологии

Ежемесячное обслуживание автотестов займет около 6-8 часов в месяц при поломках тестов и дальнейших их исправлений.
