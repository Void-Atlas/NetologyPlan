# 2 Перечень используемых инструментов с обоснованием выбора.

- **Selenide WebDrive + Java 11+**

**Обоснование**: классика, стабильная работа с динамическими SPA, хорошая поддержка сообществом,
много готовых практик по работе с формами и модалками.
- **JUnit 5+**
   
**Обоснование**: удобные фикстуры, parametrize, хорошая отчётность (особенно с allure/gradlew).
- **Allure** — система отчётности.
   
**Обоснование**: красивые отчёты с шагами, скриншотами, логами, видео — удобно показывать заказчику/менеджеру.
- **BrowserStack/SauceLabs/Selenoid** — облачная ферма браузеров.
  
**Обоснование**: если нужно проверить много комбинаций ОС, браузер и версию.
- **IntelliJ IDEA Ultimate** - удобный инструмент разработки.
- **Docker Compose** (если дадут доступ к БД)
- **Faker** для генерации случайных пользователей

# 4 Перечень и описание возможных рисков при автоматизации.

- **Риск 1**. Отправка реальной заявки на продакшен (высокий)
Последствия: спам в CRM, негатив от отдела продаж, блокировка IP.

- **Риск** 2. Изменение структуры сайта / селекторов (средний-высокий)
Последствия: падения тестов после каждого релиза.

- **Риск** 3. Возможны ложные срабатывания тестов. (низкий)
Последствия: проверка ошибок в тесте тратит ресурсы тестирования.

- **Риск 4**. Возможны затруднения по поиску локаторов элементов страницы для тестирования.
Последствия: сроки написания автотестов увеличатся.

# 5 Перечень необходимых специалистов для автоматизации.

- **Automation QA Engineer** — 1 человек (основной исполнитель: написание, отладка и поддержка автотестов).
- **Manual QA Engineer** / **Тимлид тестирования** — 1 человек (следит за исполнением, уточнение требований, проверка актуальности сценариев, ревью тестов)
